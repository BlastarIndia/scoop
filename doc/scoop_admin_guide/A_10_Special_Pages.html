A..10 Special Pages

The Special Pages Admin Tool is where you can edit and manage the ``special'', or static pages, such as the FAQ and other pages that don't change often or need comments. To use this Admin Tool, you must have the edit_special perm (A.12.27) active for your group.

To edit a special page, first select the desired page from the drop-down box then click the ``Get Page'' button. The fields below will be filled in with the appropriate data.

Delete this page
This checkbox, if checked, will cause the page to be deleted when saved. It is only displayed when a page is loaded into the form.
Page ID
is a short alphanumeric string used to uniquely identify the special page. This shouldn't have any spaces in it, as it's used primarily in the URL.
Title
is the full title of the page, displayed in the page header.
Description
is not used anywhere else, and is just to show the admin(s) what the page is for, in a few words.
Content
is the special page itself. Any HTML markup may be used here, but full HTML header and footer information isn't required, as the contents of this textarea are inserted into the |CONTENT| special block in a page template.

If the upload_content perm (A.12.55) is set, an upload field appears below the content textarea. If a file name is present in that field, the contents of that file are inserted into the content textarea when the ``Save Page'' button is pressed.
